HTML

  • HTML is used to build the "frame" of a webpage.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The body element is an example of a semantic HTML element. In semantic HTML elements, the name of the element describes its content and the role it plays on the webpage.
  • A link element creates the connection to the external file.
  • The rel attribute specifies the relationship between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external resource.

CSS

  • CSS provides the color and styling for our webpage.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline CSS allows a developer style an element directing in the HTML file.
  • Inline styles apply to a specfic HTML tag, using a style attribute with a CSS rule to style a particular page element only. They are useful for quick and permanent changes but are less flexible than external and internal style sheets.
  • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • The rules of an internal style sheet applies to all tagged page elements of that page.
  • External style shet is the most preferred as it allows developers keep their CSS rules in a separate file. This allows for flexibility and makes it easer to edit the CSS rules.
  • CSS rules consist of a selector and at least one delcaration. A delcaration contains the property and value associated with that property.
  • Margin is how much space we want around the outside of an element.
  • Padding is how much space we want around the inside of an element.
  • A class attribute allows us to add styles to certain elements, but all the elements.
  • CSS stylesheets are linked in the head element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • always work on a feature branch
  • It is important to frequently sync the local version of your code with the remote version in GitHub. Working on an outdated version can lead to big sync problems.
  • git pull origin main: syncs our local branch with the remote branch in GitHub
  • git add -A: adds our changes to the staging area
  • git commit -m "Enter a short descrpitive message here": commits our changes in our staging area to our repository

JavaScript

  • JavaScript makes a static HTML/CSS webpage interactive.
  • Think of JavaScript as all the things we can interact with in our house.
  • A variable is a named container that allows us to store data in our code.
  • JavaScript files are linked in the body element.
  • Control flow is the order in which a computer executes a script. JavaScript runs code sequentially.
  • Conditional statements allows us to interrupt control flow.
  • An array is variable that represents a group of related data.
  • For loops consist of the keyword 'for' followed by three-part conditional statement, then code block.
